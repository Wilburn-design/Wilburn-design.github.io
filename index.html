前端好用的UI库和css样式库

发表于 2022-05-16 更新于 2022-07-05 分类于 前端 ， UI库 ， css库

一、Web端

  • ElementUI

之前在开发Vue2.x的时候使用的ElementUI，样式简介好看，支持vue2.x，各种组件的属性以及API方法较为完善。

详情见：ElementUI

  • ElementPlus

使用Vue3.2和TypeScript进行开发，支持setup语法糖格式，提供完整的类型定义文件。并使用 Composition API 降低耦合，简化逻辑。并在原来的UI基础上进行了体系化的改进。

详情见：ElementPlus

  • WindiCSS

对使用过Tailwind的开发者来说，WindiCSS就可以说是Tailwind的替代，完美兼容Tailwind v2.0。我认为在这里几乎包含了开发者能用到的大部分CSS库，而且方便快速，与Element系列可以联用，兼容性好。

二、移动端

  • Vant

之前在给学校做实验室移动应用小程序时使用的Vant + Uniapp，觉得VantUI这个框架足够轻量，平均组件体积小于1KB，提供的UI样式也很多且稳定，有兴趣的兄弟可以使用试试。

详情见：Vant Weapp

  • ColorUI

与WindiCSS一样，ColorUI是一个css类库，视觉效果非常好看，在众多框架中脱颖而出，与Vant可以联用。

详情见：ColorUI

TIMESTAMP和DATETIME的区别

发表于 2021-09-10 更新于 2022-07-04 分类于 数据库

TIMESTAMP

  • 格式：YYYY-MM-DD HH:MM:SS

  • 范围：1970-01-01 00:00:01 UTC 到 2038-01-19 03:14:07 UTC

  • 存储：4字节

  • 可以设置default current_timestamp 默认获取当前时间进行存储，设置on update current_timestamp 自动维护更新时间。

  • 将服务器时间转化成UTC来存储，返回的时间会随着数据库的时区不同而发生改变。

DATETIME

  • 格式：YYYY-MM-DD HH:MM:SS
  • 范围：1000-01-01 00:00:00 到 9999-12-31 23:59:59
  • 存储：8字节
  • 存储服务器当前的时区，返回的时间与时区无关

Date、LocalDateTime之间的区别

发表于 2021-08-04 更新于 2022-07-04 分类于 Java

一、java.util.Date

  • 可读性差。

  • 由于Calendar为共享变量，在多线程并发下会产生线程安全问题。

二、java.time.LocalDateTime

可读性高，线程安全。

三、对应MySQL数据库字段类型

  • java.util.Date和java.time.LocalDateTime都对应着MySQL中的datetime/timestamp。
  • java.util.Date有时区，java.time.LocalDateTime无时区。

四、Web项目中格式化输入输出注解

  • @DateTimeFormat为格式化输入注解
  • @JsonFormat为格式化输出注解
